# bisq-burningman-dump

Headless CLI app that boots the same modules as `bisq-seednode`, syncs the DAO state via P2P,
dumps the latest burningman receiver addresses (one address per line, with owner name + share
metadata as comments above each entry) to a `.conf` file in the app data dir, then shuts down.

Output file: `<appDataDir>/burningman_addresses.conf`.

## Build

```bash
./gradlew :burningman-dump:installDist
```

Launcher: `burningman-dump/build/app/bin/bisq-burningman-dump`.

A standalone tar is also produced under `burningman-dump/build/distributions/`.

## Run

The app forces `--fullDaoNode=false` (lite DAO mode, no Bitcoin Core RPC) unless the caller passes
`--fullDaoNode` explicitly. This avoids reading `fullDaoNode` from the desktop's persisted
preferences (which would also silently mutate them if RPC creds are missing).

DAO state syncs over P2P. The app exits after the first `onParseBlockChainComplete`, so a fresh
data dir takes longer than a warm one.

### Reuse an existing Bisq desktop data directory

The desktop app's default `appName` is `Bisq` and its data dir lives in the platform default
location (e.g. `~/.local/share/Bisq` on Linux, `~/Library/Application Support/Bisq` on macOS).
Point this app at that dir by passing matching `--appName` (and `--appDataDir` if non-default):

```bash
./bisq-burningman-dump --appName=Bisq
```

Or with an explicit path:

```bash
./bisq-burningman-dump \
    --appName=Bisq \
    --appDataDir=/home/me/.local/share/Bisq
```

### Standalone (own data dir)

```bash
./bisq-burningman-dump --appDataDir=/tmp/bm-dump
```

### Flags

App-specific:

- `--include-legacy-burningmen` — also emit the two legacy burningmen (DPT + BTC fees). Default
  off. Accepts `--include-legacy-burningmen` (implicit `true`) or
  `--include-legacy-burningmen=true|false`.

Inherited from core `Config` (selection):

- `--appName=<name>` — selects the data dir name (`Bisq` for desktop, `bisq_seednode` for seeds,
  default `bisq_burningman_dump`).
- `--appDataDir=<path>` — explicit data dir override.
- `--baseCurrencyNetwork=BTC_MAINNET|BTC_TESTNET|BTC_REGTEST` — defaults to mainnet.
- `--maxConnections=N` — peer connection cap (core default `12`).
- `--useTorForBtc=true|false`, `--socks5ProxyBtcAddress=...`.

Full list: `./bisq-burningman-dump --help`.

## Output format

One bitcoin address per line. Each address is preceded by `#` comment lines describing the owner
and burningman metadata. Blank lines separate entries. Designed to be trivially iterable: skip
blank lines and lines starting with `#`, take everything else as an address.

```conf
# Bisq burningman receiver addresses
# Network: BTC_MAINNET
# Chain height: 871234
# Generated at: 2026-05-08T12:34:56Z
# Include legacy: false
# Format: one bitcoin address per line. Lines starting with '#' or blank lines
# are comments / separators and describe the entry on the following non-comment line.
# Generated by bisq-burningman-dump.

# owner: alice-contributor
# burnShareCapped=4.2100% burnShare=5.1300% compShare=2.4000% burnedSat=12345678 compSat=9876543 addrValid=true
bc1qxyz...

# owner: bob
# burnShareCapped=1.0500% burnShare=1.0500% compShare=0.7800% burnedSat=234567 compSat=345678 addrValid=true
# alt compensation addresses: [bc1q..., bc1q...]
bc1qabc...
```

With `--include-legacy-burningmen` the two legacy burningmen are appended at the end:

```conf
# owner: Legacy Burningman (DPT) (legacy)
# burnShareCapped=0.0000% burnShare=0.0000% compShare=0.0000% burnedSat=0 compSat=0 addrValid=true
38bZBj5peYS3Husdz7AH3gEUiUbYRD951t
```

Entries are sorted by `cappedBurnAmountShare` descending. Candidates without a receiver address
are skipped (they cannot receive funds).

### Example: parse the file in a shell pipeline

```bash
grep -v '^#' burningman_addresses.conf | grep -v '^$'
```
